# 枫之谷 Artale 找王与挂机助手 - 需求与架构文档 (Requirements & Architecture)

本文档旨在为下一阶段的调试与开发工作提供清晰的指南。整理了当前版本的全部核心需求点、WPF 辅助助手及 Python 后端的各 Tab（功能模块）设计说明，以及当前版本存在的技术痛点。

---

## 📌 项目基本情况
本项目是一个双端（C# WPF 客户端 & Python 后端 + Web 前端）的枫之谷《冒险岛阿尔泰（Artale）》辅助挂机与找王工具：
- **主要用途**：挂机打怪、按键 Buff 轮播、路线巡逻、防封安全检测（经验超时、符文警报）、自动找王（击杀 10 只怪判定、Boss 出现警告）。
- **当前最新代码状态**：已成功上传至 GitHub 仓库 `Artalebuffpro` 的 **`mxdboss`** 分支。

---

## 📋 核心需求点汇总 (Requirements)

1. **小怪击杀统计（找王计数）**
   - 玩家换线找王时，必须先在当前线击杀 10 只小怪才会触发 Boss 提示。
   - 工具需要通过 OCR 或特定算法检测屏幕右下角战斗日志中的“已獲得經驗值”变化，智能递增计数 `X / 10`。
   - 当计数满 10 只时，UI 变绿；当出现 Boss 时，UI 变红。
   
2. **Boss 刷新警报**
   - 当游戏画面中央出现特殊黄字提示 **“能感受到邪惡的氣息。”** 时，工具必须立即侦测并报警（闪烁红字提醒、播放警报声）。

3. **快捷键控制（换线一键归零）**
   - **`F8`**：重置计数（换线后一键将计数归零，清除 Boss 警告状态）。
   - **`F9`**：切换悬浮窗的锁定/解锁状态（解锁时可以拖动悬浮窗，锁定后变为完全鼠标穿透）。
   - **`F10`**：暂停/继续监测。
   - **`F11`**：除错截图（抓取当前 OCR 检测区域并保存，便于分析）。

4. **安全保护与超时机制**
   - 持续监控当前经验值。若在设定时间内（如 5 分钟）经验值没有发生任何增加，说明角色被怪卡死、断线或被封禁，立即执行保护性挂起、暂停挂机或关闭游戏。

## 🎛️ C# WPF 客户端 NavigationView (各 Tab 功能划分)

根据 WPF 客户端 `MainWindow.xaml` 的布局设计，系统共包含以下 8 个核心功能 Tab 页：

| # | 导航项 Content (Tag) | 对应功能模块 | 核心工作逻辑与作用 |
|:---:|:---|:---|:---|
| **1** | **按键 / 技能** (`keys`) | **Buff 按键轮播配置** | 用于配置各种技能、药水等按键的自动释放逻辑。支持常规单点、长按、随机冷却时间波动，以及独占/互斥按键（如大招或特殊状态技能释放时，临时挂起其他按键）。 |
| **2** | **巡逻** (`patrol`) | **自定义挂机路线动作链** | 配置挂机移动路线。支持动作序列（向右走 X秒 -> 停顿 Y秒 -> 向左走 Z秒），并支持定时循环抢占/打断组（每隔 N秒强制打断常规移动，去放特定技能或走特定路径）。 |
| **3** | **预设** (`presets`) | **配置存档管理** | 用于加载、保存、导出或删除不同职业/地图的挂机配置文件（如 `弩.json`）。 |
| **4** | **经验** (`exp`) | **EXP 时速统计与防呆** | 核心监控区。配置经验条区域位置，显示本次累计获取经验值、每分钟/每十分钟的经验时速。设置超时秒数阈值，若无经验增加则触发保护警报。 |
| **5** | **找王** (`boss`) | **Boss 召唤计数器** | 专门针对阿尔泰服找王。统计击杀进度，检测 Boss 特殊字样警报，并控制悬浮窗的显隐、计数、以及颜色（满10绿，Boss红）。 |
| **6** | **缩放** (`resize`) | **窗口比例适配** | 针对不同电脑分辨率（如 1080p、2K、4K）调整游戏窗口抓取范围，保证 OCR 坐标能够等比例缩放和对齐。 |
| **7** | **符文 / 警告** (`rune`) | **反外挂检测防护** | 监控画面中是否出现需要手动解除的符文、测谎仪或红字系统警告，进行弹窗/声音大分贝警报提示。 |
| **8** | **全局安全** (`settings`) | **系统配置与授权激活** | 软件激活校验（RSA HWID机器码）、抓图模式切换（如 Win32 传统抓图与 WinRT DWM 硬件加速抓图）、系统级运行参数。 |

## 🛠️ 下一阶段待调试的痛点问题 (Debug Objectives)

1. **经验区域截图不显示**
   - **表现**：在 Web 仪表盘 (`frontend.html`) 的预览区中，`logCropImg` 和 `expCropImg`（战斗日志区及经验值区域截图）显示为空白或无法成功加载。
   - **分析**：可能是截图转换 Base64 后的 SSE 数据流推送格式不正确、Tkinter 与 HTTP 服务线程冲突，或者截图模块由于 DPI 缩放（例如在 4K 屏幕下）截取到了错误的黑色区域。
   
2. **软件崩溃卡死（Freeze & Crash）**
   - **表现**：在开启监控一段时间，或者拖拽窗口后，辅助软件无响应卡死或崩溃退出。
   - **分析**：
     - **线程阻塞**：PyInstaller 编译后，OCR 处理（Windows Native OCR）或 `ImageGrab.grab()` 重复抓取操作如果运行在主 UI 线程或未受限制的后台线程中，可能会导致 GIL 锁死或 Tkinter 主循环无响应。
     - **异步队列泄露**：推送数据到 SSE 时，如果连接断开而队列继续写入，可能导致内存溢出或网络线程死锁。
